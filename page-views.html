About This Feature

The Historical Balance feature is designed for migrating existing COC (Compensatory Overtime Credit) balances from your old system into the current system.

Migration Validation Rules:
  1. Monthly Cap: COC Earned 40 hours per month
  2. Used vs Earned: COC Used must be COC Earned
  3. Issuance Date: Must be between last day of earned month and last day of the month after (e.g., Jan 31 to Feb 29 for January, or Dec 31 to Jan 31 for December)
  4. Total Balance Cap: Adding remaining hours cannot exceed employee's 120-hour total balance
  5. No Duplicates: Cannot log multiple historical balances for the same employee + month + year combination. Prevents double-entry

  • Continuous Use: Up to maximum of 5 consecutive days per single availment
  • Staggered Use: Can be used on staggered basis within the year

Help & Instructions

About This System
CompTime Tracker - COC Management System

This system is designed to manage Compensatory Overtime Credits (COC) for employees. It tracks overtime work, generates certificates, and manages CTO (Compensatory Time Off) applications with full audit trails and expiry management.

Key Features:

  • Overtime logging and certification
  • Automatic COC expiry tracking (1 year from issuance)
  • CTO application with FIFO (First In, First Out) deduction
  • Employee ledger with complete transaction history
  • Professional Excel export for reporting
  • Real-time dashboard and alerts
Quick Start Guide
1. Add Employees

Go to Employees menu Click Add Employee Fill in employee details (First Name, Last Name, Office, Position)

2. Log Overtime

Go to COC Log Overtime Select employee and month/year Enter overtime details (Date, Day Type, Time In/Out) Save

3. Generate COC Certificate

Go to COC Generate Certificate Select employee and month Click "Generate Certificate" Certificate with PDF will be created

4. Apply for CTO

Go to COC CTO Application Select employee and filing date Choose inclusive dates Select whole or half day per date Submit

5. View Employee Ledger

Go to COC View Employee Ledger Select employee View all transactions (COC earned and CTO used) Export to Excel

6. Check Dashboard

The Dashboard shows: Total COC hours, Active employees, Uncertified logs, and COC expiring in 30 days

Detailed Instructions

Adding Employees:

  1. Click Employees in the navigation menu
  2. Click the Add Employee button
  3. Fill in: First Name, Last Name, Middle Name (optional), Suffix (optional)
  4. Select Office and Position from dropdowns
  5. Set Status (Active/Inactive)
  6. Click Save

Editing Employees: Click the pencil icon next to the employee name, modify details, and save.

Note: Employee IDs are auto-generated (e.g., EMP_001, EMP_002)

  1. Go to COC Log Overtime
  2. Select the employee from the dropdown
  3. Choose the Month and Year
  4. Click Add Entry for each overtime day
  5. Fill in: Date, Day Type (Regular/Weekend/Holiday), Time In/Out (AM and PM)
  6. System automatically calculates total hours and COC earned
  7. Click Save All Entries

Important: Overtime logs must be certified before COC can be used!

By Employee (Individual):

  1. Go to COC Generate Certificate By Employee tab
  2. Select employee from dropdown
  3. Uncertified months will appear automatically
  4. Click "Generate" next to the month
  5. Enter Date of Issuance (usually first day of following month)
  6. Click Generate Certificate & PDF
  7. PDF and summary will be generated automatically

By Batch (Multiple Employees):

  1. Go to COC Generate Certificate By Batch tab
  2. Select Month and Year
  3. Click Load Employees
  4. Check the employees you want to generate certificates for
  5. Enter Date of Issuance
  6. Click Generate Selected Certificates

Note: Generated certificates are valid for 1 year from date of issuance. COC expires after this period!

Filing a CTO Application:

  1. Go to COC CTO Application Log CTO Application tab
  2. Select employee and filing date
  3. System shows available COC balance
  4. Select inclusive dates (From - To)
  5. For each date, choose: Whole Day (8 hrs) or Half Day (4 hrs)
  6. System calculates total hours to be used
  7. Click Log CTO

Important: System uses FIFO (First In, First Out) - oldest COC is used first!

Editing a CTO: Go to List of CTO Applications tab Click pencil icon Modify hours Update

Cancelling a CTO: Go to List of CTO Applications tab Click X icon Enter reason Confirm

Note: When cancelling, hours are restored to the same COC batches. If COC expired, hours are forfeited!

  1. Go to COC View Employee Ledger
  2. Select employee from dropdown
  3. View complete transaction history:
    • Earned: COC certificates issued
    • Used: CTO applications filed
    • Balance: Running balance after each transaction
  4. Click Export to Excel for formatted report

View Details: Click "View" button next to any transaction to see complete details including deductions from specific COC batches.

Cancelled CTOs: Shown with red "CANCELLED" badge and yellow background

Dashboard: Shows real-time statistics:

  • Total COC: Sum of all active COC hours
  • Active Employees: Number of employees with active status
  • Uncertified Logs: Overtime entries not yet certified
  • Expiring in 30 Days: COC hours expiring soon

Reports Page: Generate various reports:

  • Summary of COC Balances per Employee
  • CTO Applications Report
  • Expiring Credits Report

Excel Export: All reports can be exported to Excel for printing or sharing

Libraries (Master Libraries):

  • Offices: Add/Edit/Delete office names
  • Positions: Add/Edit/Delete position titles

Signatory (Master System Configuration):

  • Set the name and position of the signatory for certificates
  • This appears on all generated COC certificates

Historical Balance (Master Historical Balance):

  • Add COC balances from previous system (migration)
  • Required for employees with existing COC before system implementation

Holidays (Master System Configuration):

  • Add regular and special non-working holidays
  • Set as recurring for annual holidays (e.g., New Year, Christmas)
Important Reminders
  • COC Validity: All COC certificates are valid for 1 year from date of issuance. After expiry, unused hours are forfeited.
  • FIFO Deduction: System automatically uses oldest COC first when filing CTO applications.
  • Certification Required: Overtime logs must be certified (generate certificate) before COC can be used for CTO.
  • Cancellation: When cancelling CTO, hours are restored to original COC batches. If expired, hours are NOT restored.
  • Data Backup: Regularly export employee ledgers for backup and record-keeping.
  • Date Format: System uses MM/DD/YYYY format for date entry and display.
